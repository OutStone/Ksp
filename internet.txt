#-#-# IP adress
- IP v.4
- IP prefix
    - one for each local network

Route = a way from router to router
Routing table = table with routes and how to handle them

Forwarding = getting packet, examines the destination adress in the routing table, sends it where the table says
Hop count = number; decreases by one for each Forwarding - packet gets deleted after reaching zero

            -collection of systems under single administrative domain

        - grahp of ASNs
            - tree (mutliple interconnected ones)
                - unstream & downstream conections
                    (downstream member pays for usage of the conection)
                        -> downstream membres want peer-to-peer conections
                            (- connection goes through max. one peer-to-peer conection)
                - multiple upstream conections
                - top level nodes are peered each with every one
            - bussines side
                - no economic insentisive to make conections
                                                            from upstream  to another upstream (costs me money)
                                                            from peer      to my upstream      (costs me money)
                                                            from peer      to another peer     (computationaly hard)
                - shortest path != cheapest one
            - Border Gateway protocol
                - policy-based routing
                - global routes
                - manually configured
                - announcing only my best routes
                a) external BGP - between different ASNs
                b) internal BGP - inside of ASN
                - different rules
                - working:
                    - recieve routes
                    - filter potentialy harmfull things
                    - best path selection
                    - export policy filtering
                    - advertise routes
                - attributes
                    - AS_PATH
                        sequence of AS the info packet went through
                        newest AS is the first
                        my AS is already included -> cycle -> delete
                        shorter path lenght = better
